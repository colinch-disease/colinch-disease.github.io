# AI 회의록 요약 프롬프트 모음 — 복사해서 바로 쓰세요

![회의 요약](https://images.unsplash.com/photo-1560439514-4e9645039924?w=800&q=80)

회의록 작성이 귀찮은 이유는 "내가 다 듣고 있었는데 왜 또 써야 하지"라는 감각 때문이다. AI에게 회의 내용을 주면 정리를 대신해준다. 아래 프롬프트를 복사해서 회의 내용만 바꿔서 쓰면 된다.

## 프롬프트 1 — 기본 회의록 정리

```
아래 회의 내용을 회의록 형식으로 정리해줘.

포함 항목:
- 일시 및 참석자
- 논의 사항 (항목별)
- 결정된 사항
- 다음 액션 아이템 (담당자, 기한 포함)

[회의 내용 또는 녹취 붙여넣기]
```

## 프롬프트 2 — 핵심만 추려서 공유용

```
아래 회의 내용에서 핵심만 뽑아서 이메일로 공유할 수 있는 요약문을 써줘.
길이: 5~7줄
받는 사람: 회의에 참석하지 못한 팀원
포함: 주요 결정 사항, 각자 해야 할 일

[회의 내용 붙여넣기]
```

## 프롬프트 3 — 액션 아이템만 추출

```
아래 회의 내용에서 액션 아이템만 추출해줘.
형식: 담당자 / 할 일 / 기한 순으로 표 형식으로.

[회의 내용 붙여넣기]
```

## 프롬프트 4 — 결정이 안 된 사항 정리

```
아래 회의에서 결론이 나지 않은 사항이나 추가 논의가 필요한 항목만 정리해줘.
각 항목마다 왜 미결인지 이유와 다음 단계를 같이 써줘.

[회의 내용 붙여넣기]
```

## 프롬프트 5 — 보고용 요약 (경영진 보고)

```
아래 회의 내용을 경영진 보고용으로 요약해줘.
조건:
- 3~5문장으로 핵심만
- 의사결정 사항과 그 근거 포함
- 불필요한 과정 설명은 제외
- 결론과 다음 일정이 명확하게

[회의 내용 붙여넣기]
```

## 잘 안 되는 경우와 대처법

회의 내용이 너무 두서없거나 "어 그거 있잖아요" 같은 불완전한 대화가 많으면 AI 요약도 정확하지 않다. 이럴 땐 핵심 결정 사항만 직접 몇 가지 메모해두고 AI에게 살을 붙여달라는 방식이 낫다.

```
오늘 회의에서 결정된 사항:
1. OO 프로젝트 일정 2주 연기
2. A담당자가 클라이언트에게 연락
3. 다음 회의는 금요일 오전 10시

이 내용을 바탕으로 팀원들에게 공유할 회의 요약 이메일을 써줘.
```

---

회의록 작성은 AI가 가장 빠르게 시간을 아껴주는 업무 중 하나다. 처음 한두 번은 어색한데 패턴이 생기면 5분도 안 걸린다.
