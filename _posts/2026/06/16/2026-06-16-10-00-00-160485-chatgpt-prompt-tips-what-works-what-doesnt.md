# ChatGPT 프롬프트 잘 쓰는 법 — 잘 되는 것과 안 되는 것

![프롬프트 작성](https://images.unsplash.com/photo-1611532736597-de2d4265fba3?w=800&q=80)

ChatGPT를 처음 쓸 때 "이거 별로다"는 생각이 드는 이유의 절반은 질문 방식이 잘못됐기 때문이다. 같은 AI인데 어떻게 물어보느냐에 따라 결과가 완전히 달라진다.

## 잘 되는 패턴

**1. 역할을 부여한다**

그냥 물어보는 것보다 "당신은 OO 전문가입니다"처럼 역할을 주면 답변 수준이 올라간다.

```
# 안 좋은 예
계약서 검토해줘.

# 좋은 예
당신은 10년 경력의 계약법 전문 변호사입니다.
아래 계약서를 검토하고 불리한 조항을 찾아줘.
```

**2. 조건과 형식을 구체적으로 준다**

"~해줘"보다 길이, 형식, 포함할 항목을 명시하면 바로 쓸 수 있는 결과가 나온다.

```
# 안 좋은 예
블로그 글 써줘.

# 좋은 예
주제: 재택근무의 장단점
대상: 직장인
길이: 600~800자
형식: H2 소제목 3개 이상, 각 소제목 아래 2~3문단
톤: 친근하고 실용적으로
```

**3. 예시를 준다**

원하는 결과물과 비슷한 예시를 먼저 보여주면 방향이 맞아진다.

```
아래 예시 스타일로 제품 설명문을 써줘.

예시: "매일 아침 10분이면 충분합니다. 복잡한 설정 없이 바로 시작하는 스마트 알람."

내 제품: [제품 설명]
```

**4. 단계를 나눠서 물어본다**

한 번에 다 해달라고 하면 결과가 산만해진다. 단계별로 나눠서 검토하면서 진행한다.

```
1단계: 주제에 맞는 아웃라인만 먼저 잡아줘.
(확인 후)
2단계: 1번 섹션 내용을 써줘.
```

## 잘 안 되는 패턴

**모호한 질문:** "좋은 글 써줘", "잘 정리해줘"처럼 기준이 없으면 평범한 결과만 나온다.

**너무 많은 것을 한 번에:** "마케팅 전략 전체를 써줘"처럼 범위가 너무 크면 피상적인 답변이 온다. 쪼개서 물어봐야 한다.

**사실 확인 없이 믿기:** ChatGPT는 자신 있게 틀린 정보를 말하기도 한다. 수치, 인물 정보, 법적 내용은 반드시 다른 출처에서 확인해야 한다.

## 원하는 게 나올 때까지 수정 요청하기

한 번에 완벽한 답이 나오길 기대하지 말고, 계속 요청한다.

```
이 답변을 더 간결하게 줄여줘.
3번 항목을 더 구체적인 사례로 바꿔줘.
전체 톤을 좀 더 친근하게 바꿔줘.
```

---

프롬프트 작성에 따로 공부가 필요한 건 아니다. 구체적으로, 조건 있게, 단계별로 — 이 세 가지를 신경 쓰면 결과가 달라진다.
