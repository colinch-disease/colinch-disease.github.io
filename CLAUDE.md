## 주제 방향과 예시

이 블로그의 모든 포스트는 아래 다섯 가지 방향 중 하나에 해당해야 한다.

### 방향 1 — AI 도구 사용법 실전 예시

특정 AI 도구를 실제로 어떻게 쓰는지 보여준다. 캡처 없이도 따라할 수 있을 만큼 구체적으로.

```
주제 예시:
- "Claude 사용법 실전 예시 — 이메일·보고서·요약 각각 어떻게 쓰나"
- "ChatGPT로 업무 효율 올리는 방법 — 실제 직장인이 쓰는 5가지 패턴"
- "Perplexity 검색 활용법 — 구글 대신 쓸 수 있는 상황 정리"
- "Gemini Advanced 써보니 — 실제로 뭐가 다른지 정직하게"
- "Copilot 무료로 쓰는 법 — 모르면 손해인 기능들"
```

### 방향 2 — 직업·상황별 AI 프롬프트 모음

"나는 어떻게 쓰면 되지?"라는 질문에 바로 답이 되는 글. 복사해서 바로 쓸 수 있는 프롬프트 포함.

```
주제 예시:
- "AI 프롬프트 직업별 모음 — 마케터·기획자·디자이너·교사용"
- "ChatGPT 프롬프트 직접 써보니 — 잘 되는 것 vs 안 되는 것"
- "회의록 요약 AI 프롬프트 모음 — 복사해서 바로 쓰세요"
- "취업준비생이 AI 쓰는 실제 방법 — 자기소개서·면접 준비"
- "블로그 글쓰기에 AI 쓰는 법 — 어디까지 써도 되나"
```

### 방향 3 — AI로 부업·수익 내기 실제 사례

"AI로 돈 벌 수 있나?"에 구체적 사례와 현실적 판단을 담는다.

```
주제 예시:
- "AI로 부업하는 실제 사례 — 월 50만원 버는 사람들의 방법"
- "AI 번역으로 부업 가능한가 — 현실적으로 얼마나 벌 수 있나"
- "AI 이미지 생성으로 수익 내는 방법 — 실제로 팔리는 것들"
- "ChatGPT로 전자책 만들어 판 사람 이야기"
- "AI 도구 활용 프리랜서 — 기존보다 일 처리 속도 3배 빨라진 경험"
```

### 방향 4 — AI 뉴스·업데이트 해설

오늘 터진 AI 관련 뉴스를 일반인이 이해할 수 있는 언어로 풀어준다. "그래서 나한테 무슨 의미인가"까지 다룬다.

```
주제 예시:
- "GPT-5 출시 — 실제로 달라진 점만 골라서 정리"
- "Claude 업데이트 — 이번에 뭐가 바뀐 건지 5분 만에 파악"
- "AI 이미지 저작권 논란 — 내가 만든 이미지는 내 것인가"
- "삼성·LG 스마트폰 AI 기능 총정리 — 어떤 게 실제로 쓸 만한가"
- "구글 Gemini vs ChatGPT — 2026년 현재 어느 쪽이 나은가"
```

### 방향 5 — AI 도구 비교·추천

"뭘 써야 하나"라는 질문에 직접 답한다. 장단점보다 "이런 상황이면 이걸 써라"는 결론 중심.

```
주제 예시:
- "ChatGPT vs Claude — 솔직하게 비교해봤다"
- "AI 글쓰기 도구 추천 — 상황별로 뭘 써야 하나"
- "무료 AI 도구 모음 — 돈 안 쓰고 쓸 수 있는 것들"
- "AI 이미지 생성 도구 비교 — Midjourney·DALL-E·Stable Diffusion 차이"
- "직장인 AI 도구 추천 — 이것만 써도 충분하다"
```

```
4. 중복 여부 확인
   grep -rl "주제키워드" _posts/ --include="*.md"
```

## 글쓰기 스타일

### 언어 수준

- 기술 용어 최소화. 써야 하면 괄호 안에 쉬운 설명 추가
  - "LLM(글·그림·코드를 만들어주는 AI)"
  - "프롬프트(AI에게 내리는 지시문)"
- 문장은 짧게. 한 문장에 한 개의 생각
- 길이: 800~1,400자

### 톤

- 직접 경험한 것처럼: "써봤더니", "처음엔 별거 아닌 줄 알았는데"
- 솔직하게: 단점·한계도 숨기지 않음
- 친근하게: "~다" 체, 독자를 "당신"이 아니라 "나"의 경험으로 끌어당김

### AI 감지 표현 금지 (이 블로그 추가)

```
- "AI 기술이 발전함에 따라"

- "혁신적인 AI 솔루션"
- "패러다임의 전환"
```
